# Criterios de Validación: Estructura y Estilo Base

Para confirmar que la implementación ha sido exitosa y puede ser integrada, se deben cumplir los siguientes puntos.

## Validación Funcional

1.  **Renderizado de la Página:** El archivo `punto-limpio.html` debe cargarse en un navegador sin errores.
2.  **Existencia de Secciones:** Todas las secciones (`inicio`, `como-reciclar`, `mapa`, `guia-separacion`) deben estar presentes en el DOM.
3.  **Navegación Funcional:** Al hacer clic en los enlaces del menú de navegación, la página debe desplazarse suavemente a la sección correspondiente.
4.  **Diseño Adaptable:** El layout debe ajustarse y ser legible en una vista de escritorio y en una vista móvil (ej. 375px de ancho).

## Validación de Calidad de Código

1.  **HTML Válido y Semántico:** El código HTML debe ser semántico y no tener errores de validación.
2.  **CSS Organizado:** El CSS debe estar bien estructurado, ser legible y no contener reglas innecesarias.

## Criterios para el Merge

-   La rama de esta funcionalidad podrá ser fusionada a la principal una vez que todos los puntos de validación funcional y de calidad de código hayan sido verificados y aprobados.
